# 🛡️ SEC-编码
在 CTF（Capture The Flag）竞赛中，**编码（Encoding）** 是最基础但也最重要的概念之一。它通常出现在 Crypto（密码学）、Web（网络安全）和 Misc（杂项）题目中。

编码的核心目的是**将数据转换为一种适合传输、存储或处理的格式**，而不是为了保密（那是加密的目的）。理解编码的关键在于能够快速识别数据的特征并将其还原。

以下是 CTF 中关于编码的核心概念体系：
## 0x01 核心区别：编码 vs 加密 vs 哈希

在 CTF 中，搞清楚这三者的区别是解题的第一步：

| **概念**              | **目的**       | **特征**        | **是否需要密钥** | **可逆性**  |
| ------------------- | ------------ | ------------- | ---------- | -------- |
| **编码 (Encoding)**   | 格式转换，保证数据可用性 | 只是换种写法，算法公开   | **否**      | **完全可逆** |
| **加密 (Encryption)** | 保护数据机密性      | 输出像乱码，需要算法和密钥 | **是**      | 有密钥可逆    |
| **哈希 (Hashing)**    | 验证数据完整性      | 定长输出，不可逆（单向）  | 否          | **不可逆**  |

> **CTF 经验法则：** 如果你看到一串字符，首先尝试解码。如果解码失败或乱码，再考虑是否是加密或哈希。

## 0x02 CTF 中最常见的编码类型

#### A. Base 系列 (Base64, Base32, Base58)

这是 CTF 中出现频率最高的编码。

- **Base64:** 使用 `A-Z`, `a-z`, `0-9`, `+`, `/` 共64个字符。
    
    - **特征:** 字符串长度通常是 4 的倍数，末尾常有 1 到 2 个 `=` 号（填充符）。
        
    - _示例:_ `SGVsbG8gQ1RG` (Hello CTF)
        
- **Base32:** 使用 `A-Z` 和 `2-7`。
    
    - **特征:** 只有大写字母和数字，末尾可能有 `=`。
        
- **Base58:** 比 Base64 少了易混淆字符（如 0, O, I, l, +, /）。
    
    - **特征:** 比特币地址常用，没有等号，没有 `+` 和 `/`。
        

#### B. ASCII 与 Hex (十六进制)

计算机底层的表示方式。

- **Hex (Base16):** 使用 `0-9` 和 `A-F`。
    
    - **特征:** 看起来像 `48 65 6c 6c 6f` 或 `\x48\x65...`。通常两个字符代表一个字节（Byte）。
        
- **ASCII:** 7位或8位二进制数对应一个字符。
    
    - _考点:_ 题目可能会给你一串十进制数字（如 `97 98 99`），你需要将其转为 ASCII 字符（`abc`）。
        

#### C. URL 编码 (Percent Encoding)

主要用于 Web 题目，浏览器会将特殊字符转换为 `%` 后跟两位十六进制数。

- **特征:** 大量的 `%` 符号。
    
- _示例:_ `%20` (空格), `%3C` (<), `%2F` (/)。
    

#### D. 摩斯密码 (Morse Code) 与 盲文

常见于 Misc 题目，作为隐藏信息的手段。

- **摩斯:** 点 `.` 和划 `-` 的组合（有时会替换成其他只有两种状态的字符，如 `Ook!` 语言）。
    
- **特征:** 只有两种字符组成的长串。
    

#### E. HTML 实体编码 (HTML Entity)

用于防止 XSS 攻击或显示特殊字符。

- **特征:** 以 `&` 开头，以 `;` 结尾。
    
- _示例:_ `&#97;` (a), `&lt;` (<).
    

#### F. 其他常见编码

- **Quoted-printable:** 常见于邮件头，形式如 `=E5=BD=93`。
    
- **UUencode / XXencode:** 较古老的二进制转文本编码，常见于取证题。
    
- **Brainfuck:** 一种极简编程语言，也是一种编码形式，特征是只有 `> < + - . , [ ]` 这8个字符。

#### G. 文本混淆
这一类编码在 CTF圈（尤其是国内 CTF）被称为 **“脑洞编码”** 或 **“现代文本混淆/Esoteric Languages（深奥语言）”**。

它们本质上通常不属于标准的计算机编码（如 Base64 或 ASCII），而是基于**替换密码（Substitution Cipher）** 或 **Brainfuck 语言** 变种的趣味性混淆。它们的特点是：**视觉冲击力强、带有特定文化梗、通常有一一对应的解码工具。**

---
1. 社会主义核心价值观编码 (Core Values Encoding)

这是国内 CTF 最经典的编码之一，利用了 12 个两字词语（富强、民主、文明、和谐...）来替换字符。

- **原理：** 通常是 **Brainfuck** 语言的变种，或者是 Base 编码的变种。将 Brainfuck 的 8 个操作符（`> < + - . , [ ]`）或其他编码字符映射到这 24 个汉字上。
    
- **特征（识别方法）：**
    
    - 全文由“富强、民主、文明、和谐、自由、平等、公正、法治、爱国、敬业、诚信、友善”组成。
        
    - 看起来是一篇充满正能量的重复文本。
        
- **示例：**
    
    Plaintext
    
    ```
    公正公正公正诚信文明公正民主...
    ```
    
- **解码方式：** 必须使用专门的在线工具（搜索“核心价值观编码解码”）。
    

2. 与佛论禅 / 佛曰 (Buddha / Zen)

这类编码看起来像是一段佛经。

- **原理：**
    
    - **第一代（佛曰）：** 主要是利用“佛曰：”后面的一串汉字进行加密。本质上多为 AES 加密或特定的字符映射，通常需要一个“密钥”（Key），最常见的默认密钥是 `Tvofou` 或 `Tuofou`（即“特佛”）。
        
    - **新版（新佛曰）：** 也是一种映射，不一定需要密钥。
        
- **特征（识别方法）：**
    
    - 以 `佛曰：` 开头。
        
    - 内容包含大量生僻字或看起来像经文的字（如：摩、那、婆、加、诃）。
        
- **示例：**
    
    Plaintext
    
    ```
    佛曰：摩哈心量迦是埵...
    ```
    
- **注意：** 有时题目会提示“如是我闻”，这可能暗示开头部分或密钥。
    

3. 兽语 (Beast Speak)

- **原理：** 也是 **Brainfuck** 的变种。它将 Brainfuck 的操作符映射为“兽音”字符。
    
- **特征（识别方法）：**
    
    - 由 `wu`、`wa`、`ao` 以及 `~`（波浪号）组成。
        
    - 看起来像是动物在叫。
        
- **示例：**
    
    Plaintext
    
    ```
    ~呜哇~呜哇~呜~呜... (或者是英文形式: wuwauwa...)
    ```
    
- **解码方式：** 搜索“兽音译者”在线工具。
    

4. 阴阳怪气编码 (Yin Yang Guai Qi)

源于网络上的“阴阳怪气”说话方式，这类编码通常利用**零宽字符（Zero-width characters）** 或 **特定短语** 进行隐藏。

- **类型 A（短语替换）：**
    
    - **特征：** 只有几个特定的词重复出现，如“就这？”、“不会吧？”、“大无语事件”。
        
    - **原理：** 将这些短语映射为二进制（0 和 1），然后转 ASCII。
        
- **类型 B（零宽字符/不可见字符）：**
    
    - **特征：** 你把文本复制到记事本里，发现没东西，或者光标移动时会“卡顿”。实际上里面隐藏了 Unicode 的零宽空格（`\u200b` 等）。
        
    - **解码：** 这种通常称为 "Zero Width Text Steganography"，需要专门的提取工具。
        

5. 尊嘟假嘟编码 (O.o Encoding)

源于网络热梗“尊嘟假嘟”（真的假的），配合颜文字 `O.o`。

- **原理：** 典型的 **二进制转换**。
    
    - 通常 `O` 代表 1，`o` 代表 0（或者反过来）。
        
    - 标点符号 `.` 可能是分隔符。
        
- **特征（识别方法）：**
    
    - 全篇由 `O`、`o`、`.` 组成。
        
    - 或者直接是汉字“尊嘟假嘟”的排列组合。
        
- **示例：**
    
    Plaintext
    
    ```
    Oo.oO.OO.oo
    ```
    
- **解码思路：** 将其替换为二进制 `101001`，然后转 ASCII 码或 Base 编码。

6. AAEncode (颜文字 JS)

- **原理：** 将 JavaScript 代码转换成日式颜文字。可以直接在浏览器 Console 中运行。
    
- **特征：**
    
    Plaintext
    
    ```
    ﾟωﾟﾉ= /｀ｍ´）ﾉ ~┻━┻   //*´∇｀*/ ['_']; o=(ﾟｰﾟ)  =_=3; c=(ﾟΘﾟ) =(ﾟｰﾟ)-(ﾟｰﾟ);
    ```
    
- **解码：** 将整段代码放入浏览器控制台（Console），去掉最后的执行语句（如果有 `eval`），直接回车查看源码；或者使用专门的解码器。
    

7. JJEncode

- **原理：** 将 JavaScript 代码转换为只有符号的混乱代码。
    
- **特征：**
    
    Plaintext
    
    ```
    $=~[];$={___:++$,$$$$:(![]+"")[$],__$:++$,$_$_:(![]+"")[$],_$_:++$,$_$$:({}+"")[$]...
    ```
    
- **解码：** 同上，浏览器控制台直接运行或去混淆。
    
8. JSFuck

- **原理：** 极其极端的 JS 混淆，只使用 6 个字符 `[ ] ( ) ! +` 来编写任何 JS 代码。
    
- **特征：** 只有 `[]()!+`，长度非常长。
    
- **示例：** `[][(![]+[])[+[]]+([![]]+[][[]])...`
    

9. Brainfuck (BF)

- **地位：** 上述很多编码（兽语、核心价值观）的**鼻祖**。
    
- **特征：** 只有 `> < + - . , [ ]` 这 8 个字符。
    
- **示例：** `++++++++++[>+++++++>++++++++++<<-]>.+.`

总结与应对策略

当你在 CTF 题目中看到这类“奇怪”的文本时：

1. **不要尝试手动分析逻辑：** 这些大多是现成的算法生成的。
    
2. **辨形（Fingerprinting）：**
    
    - 看到“富强民主” $\rightarrow$ **核心价值观编码**
        
    - 看到“佛曰” $\rightarrow$ **与佛论禅**
        
    - 看到“呜哇/wuwa” $\rightarrow$ **兽音**
        
    - 看到“颜文字”且是 Web 题 $\rightarrow$ **AAEncode**
        
    - 看到“O.o” $\rightarrow$ **二进制替换**
        
3. **找工具：** 推荐收藏一些聚合类的 CTF 在线工具箱（如：**Bugku 工具箱**、**CyberChef**、**CTF-Wiki** 推荐的工具），直接搜索对应的编码名称即可“一键梭哈”。

## 0x03 如何快速识别 (Fingerprinting)

在做题时，观察**字符集**（Character Set）是判断编码类型的关键：

1. **看结尾:** 有 `=` 号 $\rightarrow$ 优先猜 Base64 或 Base32。
    
2. **看范围:** * 只有 `0-9, A-F` $\rightarrow$ Hex。
    
    - 只有 `0-9` 且数字小于 127 $\rightarrow$ ASCII 十进制。
        
    - 只有 `.` 和 `-` $\rightarrow$ 摩斯密码。
        
3. **看特殊符号:**
    
    - 有 `%` $\rightarrow$ URL 编码。
        
    - 有 `&#` $\rightarrow$ HTML 实体。

## 0x04 必备工具 (The Toolkit)

不要尝试手算，熟练使用工具是关键。

- **CyberChef (瑞士军刀):** 网页版工具，支持几乎所有编码的转换和级联操作（Magic 模块甚至可以自动识别）。
- **随波逐流:** 工具，支持几乎所有编码的转换和级联操作。
