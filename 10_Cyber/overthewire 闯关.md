## Level 0: 初始登录

### 🎯 目标
连接到游戏服务器，获取进入基础环境的权限。

### 📚 知识点预习
- **SSH (Secure Shell)**: 用于加密远程登录到另一台计算机的协议。
- **端口 (Port)**: 服务器上特定服务的入口。OverTheWire 的 SSH 端口通常是 `2220`。

### 💡 引导思路
1. 使用终端工具（如 WSL, PowerShell 或 Termius）。
2. 输入 SSH 命令，格式通常为：`ssh 用户名@地址 -p 端口`。
3. 第一次连接时会提示“密钥指纹确认”，输入 `yes`。

### 🛠️ 实验过程
```bash
# 执行以下命令登录
ssh bandit0@bandit.labs.overthewire.org -p 2220
# 密码通常是: bandit0
```

## Level 0 -> Level 1: 寻找 readme

### 🎯 目标

在主目录（home directory）中找到一个名为 `readme` 的文件，并获取其中存储的密码。该密码将用于登录 `bandit1`。

### 📚 知识点预习

- **`ls` (List)**：列出当前目录下的文件和文件夹。
    
    - 常用参数 `-l` 可以查看文件的详细信息（权限、大小、修改日期）。
        
- **`cat` (Concatenate)**：将文件内容打印（输出）到终端。这是查看文本文件内容最常用的命令。
    
- **主目录 (`~`)**：登录后默认所在的文件夹。在 Linux 中，你可以通过输入 `cd ~` 或直接输入 `cd` 回到这里。
    

### 💡 引导思路

1. **确认文件存在**：首先需要看看当前目录下到底有些什么。
    
2. **查看内容**：一旦确认了 `readme` 文件的存在，使用读取命令将其内容“倒”在屏幕上。
    
3. **保存凭据**：屏幕上显示的那个长字符串就是下一关的密码。
    

### 🛠️ 关键操作记录

Bash

```
# 列出当前目录的文件
ls

# 读取 readme 文件的内容
cat readme

# 退出当前登录（准备用新密码登录下一关）
exit
```

### 🧠 经验笔记    
- **Linux 敏感性**：Linux 是区分大小写的。`readme` 和 `ReadMe` 是两个完全不同的文件。

## Level 0 -> 1: 基础文件读取
**目标**: 读取当前目录下的 `readme` 文件。
- **核心知识点**: `ls` (列出文件), `cat` (显示内容)。
- **操作**: `cat readme`

## Level 1 -> 2: 处理特殊文件名
**目标**: 读取名为 `-` 的文件。
- **核心知识点**: 破折号 `-` 在 Linux 命令中通常表示“标准输入/输出”。
- **解题思路**: 必须通过**路径前缀**来告知命令这是一个文件，而不是参数。
- **关键命令**: `cat ./-`

## Level 2 -> 3: 处理空格文件名
**目标**: 读取名为 `spaces in this filename` 的文件。
- **核心知识点**: **引号包裹**或**转义字符**。
- **解题思路**: Linux 以空格分隔参数，处理带空格的文件名有三种方法：
    1. 双引号: `cat "spaces in this filename"`
    2. 反斜杠转义: `cat spaces\ in\ this\ filename`
    3. Tab 键自动补全（最推荐）。

## Level 3 -> 4: 隐藏文件处理
**目标**: 读取 `inhere` 目录下的隐藏文件。
- **核心知识点**: 以 `.` 开头的文件是隐藏文件。
- **关键命令**: `ls -a` (all) 才能看到隐藏文件。
- **操作**: `cd inhere` -> `ls -a` -> `cat .hidden_file_name`

## Level 4 -> 5: 文件类型鉴定
**目标**: 在 `inhere` 目录下众多二进制文件中找到唯一的 ASCII 文本文件。
- **核心知识点**: `file` 命令。在 Linux 中，后缀名不代表文件类型，`file` 命令通过查看文件头信息判定真实属性。
- **操作**: `file ./*` 观察输出，寻找显示为 `ASCII text` 的文件。

## Level 5 -> 6: 多条件大规模搜索
**目标**: 在 `maybehere` 目录下寻找：1. 人类可读；2. 大小为 1033 字节；3. 不可执行的文件。
- **核心知识点**: `find` 命令的组合使用。
- **关键命令**: `find . -type f -size 1033c ! -executable`
    - `-size 1033c`: `c` 表示字节 (bytes)。
    - `!`: 表示取反。

## Level 6 -> 7: 全系统搜索与过滤
**目标**: 在全服务器寻找属于用户 `bandit7`、组 `bandit6` 且大小为 33 字节的文件。
- **核心知识点**: 用户/组权限过滤及错误重定向。
- **解题思路**: 搜索根目录会产生大量 `Permission denied` 报错，需使用 `2>/dev/null` 屏蔽它们。
- **关键命令**: `find / -user bandit7 -group bandit6 -size 33c 2>/dev/null`

## Level 7 -> 8: 文本内容过滤
**目标**: 在 `data.txt` 中找到单词 `millionth` 旁边的密码。
- **核心知识点**: `grep` (文本搜索)。
- **操作**: `grep "millionth" data.txt`

## Level 8 -> 9: 统计与唯一性
**目标**: `data.txt` 中某行只出现过一次，获取该行。
- **核心知识点**: `sort` (排序) 与 `uniq` (去重)。
- **注意**: `uniq` 只能处理相邻的重复行，所以必须先排序。
- **关键命令**: `sort data.txt | uniq -u`

## Level 9 -> 10: 提取二进制中的字符串
**目标**: 在二进制文件 `data.txt` 中寻找几个 `=` 符号后的可读字符串。
- **核心知识点**: `strings` 命令。用于从非文本文件中提取可显示的字符串。
- **操作**: `strings data.txt | grep "=="`

## Level 10 -> 11: Base64 编码处理
**目标**: 解码 base64 格式的文本文件。
- **核心知识点**: 数据编码转换。Base64 常用于在文本协议中传输二进制数据。
- **关键命令**: `base64 -d data.txt`
